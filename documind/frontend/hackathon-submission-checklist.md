# Hackathon Submission Checklist

This checklist is for a non-breaking delivery pass before final submission.

## 1. Product Demo Videos

- [ ] Record Overview route (`/`) walkthrough.
- [ ] Record Instances route (`/instances`) walkthrough.
- [ ] Record Knowledge Bases route (`/knowledge-bases`) walkthrough.
- [ ] Record Resources route (`/resources`) walkthrough.
- [ ] Record Search route (`/search`) walkthrough.
- [ ] Record Ask route (`/ask`) walkthrough.
- [ ] Record Chat route (`/chat`) walkthrough.
- [ ] Record System route (`/system`) walkthrough.
- [ ] Upload videos and capture shareable links.

## 2. Documentation Pass

- [ ] Update `tomorrow.md` with final completed items.
- [ ] Update `integration-todo.md` with final verification status.
- [ ] Add a short judge demo script (2-3 minute flow).
- [ ] Confirm known limitations are documented (`eslint` unavailable, manual flows).

## 3. Deployment

- [ ] Deploy latest frontend build to target hosting.
- [ ] Verify all 8 routes return expected response on deployed URL.
- [ ] Run one live sanity flow on deployed environment (create/update + toast).
- [ ] Capture final production URL for submission form.

## 4. Submission Package

- [ ] Final architecture summary (problem, approach, stack, trade-offs).
- [ ] Links bundle: repo, deployed app, videos, docs.
- [ ] Screenshots (hero screen + 2 feature screens).
- [ ] Complete and submit hackathon form.
- [ ] Save confirmation receipt/screenshot of submission.
